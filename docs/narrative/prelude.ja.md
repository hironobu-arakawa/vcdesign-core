# Prelude — なぜ、システムは「負債」に変わるのか
VCDesign（Value Continuity Design）。
それは「どう作るか」ではなく、作られた後の価値を、どう守り抜くかを扱う設計思想です。
多くのプロジェクトでは、「完成」がゴールになります。
仕様を満たし、テストを通過し、リリースを祝う。
そこで成功とみなされます。
しかし数年後、現場に残るのは厳しい現実です。
なぜこの設計になっているのか、誰も説明できない。
動いてはいるが、誰も中身を触りたがらない。
かつての意図は忘れ去られ、システムはただの「維持すべき重荷」になる。
システムは生き残った。
だが、価値は続いていない。
そして厄介なことに、
その価値が失われた瞬間を、誰も見ていない。
それは完成でも、障害でもない。
日々の運用という時間の中で、
少しずつ、静かに起きるからです。
VCDesign は、
その「誰も見ていない価値の断絶」を、
構造として扱おうとする試みです。
ここでいう構造とは、
カチカチに固めることではありません。
時間が経っても、人が入れ替わっても、
「なぜそうしたのか」という意図と責任が、次へと手渡され続けるための仕組みのことです。
